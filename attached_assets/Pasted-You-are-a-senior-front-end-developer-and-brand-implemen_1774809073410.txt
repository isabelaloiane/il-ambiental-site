You are a senior front-end developer and brand implementation specialist. Your task is to update the entire existing website for IL Engenharia e Consultoria Ambiental by applying the official brand identity. All brand assets are located in the /assets folder of this project.

## PHASE 0 — ASSET DISCOVERY AND ANALYSIS

Before making any code changes, scan and catalog the entire /assets folder:

### Step 0.1 — Identify all files in /assets
List every file in /assets and its subfolders. Categorize each file as one of:
- LOGO_PRINCIPAL: The file that has been explicitly renamed to indicate it is the main/primary logo (look for filenames containing words like "principal", "main", "primary", "logo-principal", "horizontal", or the largest/most complete horizontal logo with the monogram + "Engenharia & Consultoria Ambiental" text)
- LOGO_VARIANT: Other logo versions (submarca, vertical, stacked, etc.)
- ICON: The circular monogram symbol only (the "il" mark inside a circle, without accompanying text — smallest/simplest mark). Use for favicon.
- PATTERN: Repeating brand pattern files
- MANUAL: PDF files containing brand guidelines (manual de marca, identidade visual)
- OTHER: Any other asset

Print the catalog as a comment at the top of the main CSS file:
```css
/*
 * === IL AMBIENTAL BRAND ASSETS MAP ===
 * Logo principal: /assets/[filename]
 * Logo variant 1: /assets/[filename] — [description]
 * Logo variant 2: /assets/[filename] — [description]
 * Favicon source: /assets/[filename]
 * Pattern: /assets/[filename]
 * Manual: /assets/[filename]
 * =====================================
 */
```

### Step 0.2 — Read the brand manual PDF
Open and analyze any PDF file in /assets that appears to be a brand manual or identity guide. Extract from it:
- Exact HEX color codes
- Font family names
- Logo usage rules (protection area, prohibited uses, approved backgrounds)
- Approved color contrast combinations
- Any spacing or composition guidelines

Use these extracted values as the single source of truth for every design decision below. If a value in this prompt conflicts with what the manual says, the manual wins.

### Step 0.3 — Determine asset assignments

Based on the catalog:

**Header logo:** Use LOGO_PRINCIPAL — the file explicitly renamed as the main logo. This is the horizontal version with the circular monogram + "Engenharia & Consultoria Ambiental" text beside it. Use for ALL header implementations across all pages.

**Favicon:** Use the ICON file — the smallest mark (circular monogram only, no text). If no standalone icon file exists, use the submarca (circular badge version). If neither exists, use LOGO_PRINCIPAL as fallback.

**Footer logo:** Use LOGO_PRINCIPAL. On the dark footer background, apply a CSS filter to render it in beige/light tones. If a separate light-colored logo variant exists in /assets (white or beige version), use that file directly instead of a filter.

**Dark section logo (hero, CTA blocks):** Same logic as footer — use a light variant file if available, otherwise apply CSS filter to LOGO_PRINCIPAL.

**Pattern:** If a PATTERN file exists, it can be used as a subtle decorative background element at very low opacity (3-5%) in accent sections.

## PHASE 9 — SCROLL ANIMATIONS AND HEADER SCROLL

Add to the main JavaScript file (or create one if none exists):
```javascript
// Scroll-triggered animations
const observer = new IntersectionObserver(
  (entries) => entries.forEach((e) => { if (e.isIntersecting) e.target.classList.add('is-visible'); }),
  { threshold: 0.12 }
);
document.querySelectorAll('.animate-on-scroll').forEach((el) => observer.observe(el));

// Header scroll shadow
const hdr = document.querySelector('header') || document.querySelector('.header') || document.querySelector('nav');
if (hdr) window.addEventListener('scroll', () => hdr.classList.toggle('scrolled', window.scrollY > 40));
```
```css
.animate-on-scroll {
  opacity: 0;
  transform: translateY(28px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}
.animate-on-scroll.is-visible { opacity: 1; transform: translateY(0); }
.animate-on-scroll:nth-child(2) { transition-delay: 100ms; }
.animate-on-scroll:nth-child(3) { transition-delay: 200ms; }
.animate-on-scroll:nth-child(4) { transition-delay: 300ms; }
```

Add the class `animate-on-scroll` to all cards, section headers, and major content blocks. Do NOT add it to the header, footer, or WhatsApp button.
