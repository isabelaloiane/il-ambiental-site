## STEP 2 — VERIFY AND FIX ALL INTERNAL LINKS

Scan every HTML file and verify:

1. **Navigation links** — Every `<a href="...">` pointing to an internal page must use correct relative paths:
   - Links to home: `href="index.html"` or `href="/"`
   - Links to services: `href="servicos.html"`
   - Links to contact: `href="contato.html"`

2. **CSS links** — Every `<link rel="stylesheet" href="...">` must use correct relative paths. If CSS files are in a subfolder (e.g., `/css/style.css`), ensure the path is correct from each HTML file's location.

3. **JS links** — Every `<script src="...">` must use correct relative paths.

4. **Image/asset links** — Every `<img src="...">`, `background-image: url(...)`, and other asset references must use correct relative paths. No absolute file system paths (e.g., `/Users/...` or `C:\...`).

5. **Anchor links** — Any `href="#section-id"` must have a matching `id` attribute in the target element.

6. **No broken links** — Remove or fix any links pointing to deleted pages (e.g., projects page or any page that no longer exists).

## STEP 4 — VERIFY ASSET LOADING

1. Check that all images referenced in HTML and CSS actually exist in the project directory. If an image file is missing, add a comment in the code: `<!-- MISSING ASSET: filename.ext -->` but do NOT remove the reference.

2. Check that all Google Fonts imports are using HTTPS URLs and are loading correctly:
