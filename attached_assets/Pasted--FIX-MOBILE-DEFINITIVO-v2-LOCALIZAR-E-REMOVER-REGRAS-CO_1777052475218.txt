 FIX MOBILE DEFINITIVO v2: LOCALIZAR E REMOVER REGRAS CONFLITANTES
Site: ilambiental.com.br
Problema confirmado por prints reais no iPhone (390px) — header fixo de 73px sobrepõe o H1 das heroes.
 
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 
CAUSA RAIZ CONFIRMADA — LEIA ANTES DE QUALQUER AÇÃO:
 
Existem múltiplas rodadas de correções CSS mobile acumuladas no arquivo de estilos.
O resultado é um conflito de cascade onde regras antigas com `padding-top: 88px !important`
para `.hero-animated-bg` e `.page-hero` continuam vencendo porque estão posicionadas
APÓS qualquer tentativa de override anterior no arquivo.
 
A regra conflitante tem esta forma (pode estar dentro de um @media (max-width: 767px)):
 
  .hero-animated-bg,
  .page-hero {
    padding: 88px 20px 40px !important;
    /* ou */
    padding-top: 88px !important;
  }
 
Com o header fixo de 73px, padding-top de 88px dá apenas 15px de respiro — INSUFICIENTE.
O target correto é padding-top: calc(var(--nav-height) + 56px) = 129px.
 
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 
PASSO 1 — LOCALIZAR AS REGRAS CONFLITANTES NO CSS:
 
Abrir o arquivo de estilos principal (style.css, globals.css, main.css ou equivalente).
Usar Ctrl+F (busca) e pesquisar pelos seguintes termos, um por vez:
 
  Busca 1: "padding: 88px"
  Busca 2: "padding-top: 88px"
  Busca 3: ".hero-animated-bg" dentro de um bloco @media (max-width: 767px)
  Busca 4: ".page-hero" dentro de um bloco @media (max-width: 767px)
 
Para CADA ocorrência encontrada dentro de @media (max-width: 767px):
→ Verificar se define padding-top ou padding com valor fixo em px (ex: 88px, 80px, 100px, 110px)
→ Se sim: COMENTAR a linha ou o bloco inteiro usando /* ... */
→ NÃO deletar — apenas comentar para preservar o histórico
 
EXEMPLO de como comentar:
 
ANTES:
  .hero-animated-bg,
  .page-hero {
    padding: 88px 20px 40px !important;
  }
 
DEPOIS:
  /* DESATIVADO — conflito com fix mobile v2
  .hero-animated-bg,
  .page-hero {
    padding: 88px 20px 40px !important;
  }
  */
 
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 
PASSO 2 — ADICIONAR O BLOCO DEFINITIVO NO FINAL DO ARQUIVO CSS:
 
Após comentar os conflitos, ir até o FINAL ABSOLUTO do arquivo de estilos principal
e adicionar o bloco abaixo. Ele deve ser a última coisa no arquivo para garantir
precedência máxima no cascade.
 
/* ============================================================
   MOBILE FIX DEFINITIVO v2 — IL AMBIENTAL
   Adicionado ao FINAL do CSS para máxima precedência
   Resolve: header fixo 73px sobrepondo hero sections no mobile
   ============================================================ */

  /* HEROES — todas as páginas

    padding-top: calc(var(--nav-height, 73px) + 56px) !important;

SE O PROBLEMA PERSISTIR MESMO APÓS ESSES PASSOS:

Nesse caso, buscar nos arquivos de componente pelo texto "88px" ou "padding-top" dentro
de media queries e aplicar o mesmo processo de comentar + substituir com o valor
