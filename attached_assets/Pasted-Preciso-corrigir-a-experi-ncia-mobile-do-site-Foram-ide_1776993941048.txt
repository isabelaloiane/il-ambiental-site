Preciso corrigir a experiência mobile do site. Foram identificados os problemas a seguir, todos com diagnóstico técnico preciso. Aplique todas as correções sem alterar textos, estrutura de layout ou identidade visual. Apenas CSS e, onde indicado, pequenos ajustes de HTML.

PROBLEMA 1 — CRÍTICO: Hero com texto cortado (overflow)
Causa raiz: .hero-content-inner tem margin-left: 120px aplicado. No mobile (390px), isso empurra o conteúdo para fora da tela, cortando o título, o parágrafo e os botões à direita.
Fix:
css@media (max-width: 767px) {
  .hero-content-inner {
    margin-left: 0 !important;
    width: 100% !important;
    max-width: 100% !important;
    box-sizing: border-box !important;
  }
}

PROBLEMA 2 — CRÍTICO: Seção "Sobre" (about-strip) vazando da tela
Causa raiz: Os filhos diretos do .about-strip têm width: 421px e padding: 72px 76.8px — ambos excedem a largura da tela (390px).
Fix:
css@media (max-width: 767px) {
  .about-strip > div {
    width: 100% !important;
    max-width: 100% !important;
    box-sizing: border-box !important;
    padding: 40px 24px !important;
  }
}

PROBLEMA 3 — ALTO: Botões do hero lado a lado no mobile
Causa raiz: .hero-cta-group (ou .fade-3) tem flex-direction: row e gap: 14px. No mobile, os dois botões ficam na mesma linha e o segundo fica cortado ou apertado.
Fix:
css@media (max-width: 767px) {
  .hero-cta-group,
  .fade-3[class*="hero"] {
    flex-direction: column !important;
    width: 100% !important;
    gap: 12px !important;
  }

  .hero-cta-group a,
  .fade-3 a {
    width: 100% !important;
    text-align: center !important;
    padding: 15px 24px !important;
    box-sizing: border-box !important;
  }
}

PROBLEMA 4 — ALTO: Títulos de seção muito grandes para mobile
Causa raiz: Os h2 das seções foram dimensionados para desktop (44–48px) e não têm redução para mobile. No celular ficam desproporcional ao conteúdo.
Valores medidos:

Seção serviços: 44.8px
About-strip: 41.6px
Números/stats: 38.4px
Empresas clientes: 38.4px
CTA section: 48px

Fix:
css@media (max-width: 767px) {
  section h2 {
    font-size: clamp(22px, 6vw, 28px) !important;
    line-height: 1.25 !important;
  }

  .about-strip h2,
  .about-strip h3 {
    font-size: clamp(20px, 5.5vw, 26px) !important;
    line-height: 1.3 !important;
  }

  .cta-section h2 {
    font-size: clamp(22px, 6vw, 28px) !important;
    line-height: 1.25 !important;
  }
}

PROBLEMA 5 — MÉDIO: Espaçamento vertical excessivo entre seções
Causa raiz: Todas as sections têm padding-top: 80px; padding-bottom: 80px. No mobile, isso cria espaços brancos muito grandes entre blocos, prejudicando o ritmo da leitura.
Fix:
css@media (max-width: 767px) {
  section {
    padding-top: 56px !important;
    padding-bottom: 56px !important;
  }

  .cta-section {
    padding-top: 56px !important;
    padding-bottom: 56px !important;
  }

  .hero-animated-bg {
    padding-bottom: 40px !important;
  }
}

PROBLEMA 6 — MÉDIO: Footer com padding excessivo no mobile
Causa raiz: O footer tem padding: 72px 0px 36px. No mobile, 72px de padding superior é desproporcional.
Fix:
css@media (max-width: 767px) {
  footer,
  .site-footer {
    padding-top: 48px !important;
    padding-bottom: 24px !important;
  }
}
